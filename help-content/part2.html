Import and Export

You may save your process model by clicking on the download link. You can reload the file produced by dragging and dropping it back onto the page.

The model has support for importing xml files exported from PeriMeta. Again, drag and drop the file onto the page.

Wiki Integration

The model may also be loaded from and saved to a set of linked Gitit wiki pages. Each node will get its own page.

  • The URL of the page (made relative to the base URL of the wiki) becomes the name of the node.
  • A table containing a column named 'child', and optionally columns 'necessity' and 'sufficiency', is a list of edges. The 'child' column should contain an HTML link.
  • The properties for each type of node belong in another table, which has a column for each property and a single row containing the values.
  • Other content present on the page will be used to populate the node's description.

Note that you will either need to serve all pages from the same origin, or add the Access-Control-Allow-Origin header to the parent page.
